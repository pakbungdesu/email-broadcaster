
# Email Broadcaster

A lightweight, terminal-based utility designed to manage subscriber records via an interactive text user interface (TUI) and distribute automated emails via the Mailtrap API.

## 📁 Repository Structure

```text
email-broadcaster/
├── .env                  # Local environment variables (API credentials)
├── .gitignore            # Excludes logs and sensitive credentials from Git
├── broadcast.sh          # Core background email broadcasting automation script
├── interactive.sh        # TUI Subscriber Management Dashboard (Interactive)
├── output.log            # Execution and error logging output
├── README.md             # Project documentation
└── subscribers.csv       # Flat-file pipe-delimited subscriber database

```

---

## 🛠️ Features

### 🖥️ TUI Dashboard (`interactive.sh`)

Uses Linux `whiptail` dialog boxes to provide an interactive text menu for local database administration without manual formatting risks:

* **Add New Email:** Simple forms for user details, including a custom radio list selector for categories (`Personal`, `Work`, `Marketing`, `Newsletter`).
* **View All Emails:** Parses the database file to quickly display your current subscriber list.
* **Edit Email:** Safely updates existing email addresses using inline `sed` replacement.
* **Delete Email:** Instantly drops selected subscriber rows from the text file.

### 📡 Automated Broadcaster (`broadcast.sh`)

* **State Persistence:** Rewrites `subscribers.csv` line-by-line during runtime to flip status values from `pending` to `sent`, offering smooth crash recovery.
* **Intelligent Skip Logic:** Safeguards against spamming by skipping empty lines and entries already marked as `sent`.
* **Rate-Limit Throttling:** Incorporates a built-in `sleep 10` execution buffer between requests to respect Mailtrap API throughput boundaries.

---

## 📋 Data Layout

The scripts manipulate records inside `subscribers.csv` using a pipe-delimited (`|`) formatting layout:

```text
<email>|<subject>|<body>|<category>|<status>

```

> **Note:** Fresh records generated by the dashboard default to 4 fields. The automation script (`broadcast.sh`) will automatically check, append, and handle the 5th `<status>` column during live dispatch loops.

## 🚀 Getting Started

### 1. Prerequisites

Ensure your Linux system has `whiptail` installed (pre-installed on most Ubuntu/Debian environments):

```bash
# Ubuntu / Debian
sudo apt-get install whiptail

```

### 2. Configuration

Save your Mailtrap API parameters directly inside the `.env` file at the root of the project:

```bash
# .env layout
CSV_FILE="subscribers.csv"
YOUR_ID="your_mailtrap_inbox_id"
API_TOKEN="your_mailtrap_api_token"
YOUR_ID="your_mailtrap_id"

```

### 3. Execution

* **To Manage Subscribers:**
```bash
chmod +x interactive.sh
./interactive.sh

```


* **To Broadcast Pending Emails:**
```bash
chmod +x broadcast.sh
./broadcast.sh

```
